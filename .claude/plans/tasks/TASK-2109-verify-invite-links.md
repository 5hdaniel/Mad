# TASK-2109: Verify Invite Links on app.keeprcompliance.com

## WORKFLOW REQUIREMENT

**This is a verification/config task, not a code task.** The standard agent-handoff workflow applies but implementation is primarily Vercel config + manual testing.

## Goal

Ensure invite links generated by the broker portal use `app.keeprcompliance.com` (not `www.keeprcompliance.com`). This requires setting the `NEXT_PUBLIC_APP_URL` env var in Vercel and verifying the invite flow end-to-end.

## Non-Goals

- Do NOT modify application code (PR #1029 already updated the fallback)
- Do NOT change DNS or domain configuration
- Do NOT modify any other Vercel env vars

## Steps

### 1. Set Vercel Environment Variable

In Vercel dashboard (broker-portal project):
- **Settings > Environment Variables**
- Add/update: `NEXT_PUBLIC_APP_URL` = `https://app.keeprcompliance.com`
- Scope: **Production** environment
- This overrides the fallback in `broker-portal/lib/actions/inviteUser.ts:170`

### 2. Trigger Redeployment

Redeploy the production deployment so the new env var takes effect:
```bash
# Via Vercel CLI or dashboard
vercel --prod
```
Or use the Vercel dashboard "Redeploy" button on the latest production deployment.

### 3. End-to-End Verification

1. Log into `app.keeprcompliance.com` as an IT admin
2. Navigate to Team/Org settings
3. Invite a new user by email
4. Check the generated invite link:
   - **Expected:** `app.keeprcompliance.com/invite/TOKEN`
   - **Failure:** `www.keeprcompliance.com/invite/TOKEN`
5. Click the invite link — verify it loads the invite acceptance page on `app.keeprcompliance.com`

### 4. Document Results

Record verification results in the PR or sprint notes. If the invite link uses the wrong domain, investigate `inviteUser.ts` env var resolution.
